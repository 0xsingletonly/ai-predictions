## 1. Objective

Build a lightweight AI reasoning agent that selects high-signal geopolitical and macro questions on Polymarket, forms daily probability estimates from news and data ingestion, and evaluates reasoning quality against real-money market resolution.

> **Primary goal:** Evaluate reasoning quality of LLM-based probabilistic updating — not maximise P&L. Polymarket serves as the ground-truth benchmark and calibration reference.

---

## 2. Success Metrics

| Metric | Definition | v0 Target |
|---|---|---|
| Brier Score | Calibration accuracy vs resolved binary outcome (lower is better) | Beat naive 0.5 baseline |
| vs Market Baseline | Agent Brier vs Polymarket community price, validate that LLM's reasoning is non-trivial | Within 10% of market |
| Reasoning Audit Score | Manual post-mortem quality (1–5 scale) | Average ≥ 3 / 5 |
| Question Coverage | Questions tracked through full resolution | 5 fully resolved |
| Update Consistency | Days where agent updates without valid evidence trigger | < 20% of updates |

## 3. Scope

### In Scope for v0
- Question selection and portfolio management (max 5 active questions)
- Daily reasoning loop: ingest news → update probability → log rationale
- LLM integration via Kimi API (`kimi-k2.5`) for reasoning
- Polymarket integration: Gamma API for market discovery, CLOB API for price feeds
- Brier score tracking and post-resolution audit
- Dashboard UI: question monitor, probability chart, reasoning inspector

### Explicitly Out of Scope
- Automated trade execution or real money deployment
- Fine-tuning or custom model training
- Technical analysis signals (pure macro/geopolitical reasoning only)
- More than 3 topic categories simultaneously
- Mobile-responsive UI (desktop-first for v0)

---

## 4. Question Selection Criteria

All active questions must satisfy every criterion below:

| Criterion | Requirement | Rationale |
|---|---|---|
| Horizon | 2–4 weeks to resolution | Fast iteration; enough time for meaningful signal |
| Liquidity | > $50k Polymarket volume | Ensures market price is well-calibrated |
| Resolution type | Categorical outcome, unambiguous criteria | Avoids polluting Brier score with edge cases |
| Information sensitivity | Named news sources should move probability | Confirms LLM has actionable signals to parse |
| Causal legibility | Agent can pre-specify what signals to watch | Prevents post-hoc rationalisation |

**Preferred topic categories for v0:**
- Geopolitical events (Middle East, US foreign policy, sanctions)
- Central bank decisions and rates (Fed, ECB, BOJ)
- Energy and commodities (oil supply, Brent/WTI thresholds)

**Avoid:**
- Sports, entertainment, or crypto price markets
- Questions with subjective or delayed resolution criteria
- Questions with < 2 weeks of remaining market life at intake

## 6. LLM Integration

### 6.1 Model

Use `kimi-k2.5` for all reasoning steps. Do not fine-tune for v0 — structured prompting is the primary lever.

### 6.2 Daily Reasoning Pipeline

For each active question, the agent runs 4 sequential LLM calls per day:

| Step | LLM Task | Output |
|---|---|---|
| ① Evidence classification | Classify each news item: `supports_yes / supports_no / neutral / noise` | Tagged evidence list with rationale |
| ② Bull case synthesis | Given evidence, make the strongest case probability should be higher | 1–2 paragraph bull argument |
| ③ Bear case synthesis | Given evidence, make the strongest case probability should be lower | 1–2 paragraph bear argument |
| ④ Posterior + WTCMM | Weigh bull/bear, output new probability, generate `what_would_change_my_mind` | Structured JSON (see §6.3) |

### 6.3 Structured Output Schema

```json
{
  "question_id": "polymarket-condition-id",
  "date": "2026-03-14",
  "prior_probability": 0.42,
  "posterior_probability": 0.48,
  "delta": 0.06,
  "polymarket_price": 0.51,
  "divergence_from_market": -0.03,
  "key_evidence": ["Iran FM statement", "oil tanker movement report"],
  "evidence_classification": {
    "supports_yes": [],
    "supports_no": [],
    "noise": []
  },
  "bull_case": "...",
  "bear_case": "...",
  "what_would_change_my_mind": "...",
  "update_confidence": "low | medium | high",
  "reasoning_summary": "..."
}
```

> **Critical constraint:** `what_would_change_my_mind` is mandatory on every update. If the LLM cannot populate it with a specific, falsifiable condition, the pipeline flags this as an anchoring warning and surfaces it in the UI.

## 8. Dashboard UI

### 8.1 Overview

A single-page React dashboard for monitoring active questions, inspecting reasoning traces, and tracking performance. Desktop-first. Dark theme to match Polymarket's aesthetic.

### 8.2 Views

| View | Purpose | Key Components |
|---|---|---|
| Portfolio Overview | At-a-glance status of all 5 active questions | Question cards with current probability, delta, market divergence badge |
| Question Detail | Full reasoning trace for one question over time | Probability chart (agent vs market), daily reasoning log, bull/bear panels |
| Reasoning Inspector | Step-through each day's LLM reasoning chain | Evidence list with classification tags, WTCMM display, confidence indicator |
| Performance | Brier score tracking post-resolution | Brier score vs market baseline, calibration curve, resolved question history |

### 8.3 Tech Stack

- **Framework:** React (Vite)
- **Charts:** Recharts for probability time series
- **Styling:** Tailwind CSS
- **Data:** SQLite via lightweight Python FastAPI backend
- **State:** React Query for data fetching / caching

### 8.4 Key UI Behaviours

- Probability delta shown with directional colour coding: green (towards yes), red (towards no)
- Anchoring warnings surface as yellow banners when WTCMM field is flagged empty or repetitive
- Market divergence > ±15% triggers a "Market knows more?" warning in the question card
- All reasoning chains are stored and viewable historically — not just the latest day

## 9. Evaluation Framework

### 9.1 During Active Period (Daily)

- Log agent probability vs Polymarket price — track divergence over time
- Flag days where `update_confidence = high` but `delta < 0.02` (potential anchoring)
- Flag days where noise-classified evidence is dominant but probability still moves (overreaction signal)

### 9.2 At Resolution

- Compute Brier score: `(probability - outcome)²` where `outcome ∈ {0, 1}`
- Compare against Polymarket community Brier score on the same question
- Run structured post-mortem using the template below

### 9.3 Post-Mortem Template

| Question | What to Assess |
|---|---|
| Signal identification | Did the agent name the right signals to watch at intake? |
| Update appropriateness | Did it respond correctly when those signals arrived? |
| WTCMM activation | Was the `what_would_change_my_mind` condition triggered? Did the agent respond? |
| Market vs agent | At which moments was the agent better/worse calibrated than the market? |
| Failure classification | Prior anchoring / noise overreaction / missing signal / correct but unlucky |

## 11. Known Risks & Mitigations

| Risk | Impact | Mitigation |
|---|---|---|
| Question resolves abruptly overnight | Medium | Pipeline handles sudden probability jump gracefully; not treated as agent failure |
| Polymarket price embeds insider information | High | Divergence > 15%: first hypothesis is "market knows more". Surface as UI warning. |
| LLM narrative capture / anchoring | High | Adversarial bull/bear step + mandatory WTCMM + anchoring detection in Phase 3 |
| Polymarket API rate limits | Low | Daily polling only — well within free tier. Add exponential backoff. |
| Singapore jurisdiction access | Medium | Confirm Polymarket TOS compliance before any real-money deployment. v0 is read-only. |
